# Contributing

Thanks for helping improve this GreyNOC repository.

## What is Welcome

- Stars, watches, forks, comments, issues, discussions, and pull requests are welcome when GitHub permissions allow them.
- Bug reports, documentation fixes, detection logic, playbook improvements, and small focused code changes are welcome.
- Security-sensitive reports should follow SECURITY.md instead of being posted publicly.

## Ground Rules

- Keep contributions lawful, respectful, and relevant to this repository.
- Do not submit secrets, credentials, private data, malware, backdoors, destructive payloads, or code intended to harm systems.
- Do not paste exploit details or sensitive vulnerability data into public issues.
- Do not add large binary files, generated dumps, or vendored dependencies unless GreyNOC approves that first.
- Only submit work you have the right to contribute under this repository's current license.

## Change Process

1. Open an issue or pull request with a clear description of the problem and proposed change.
2. Keep pull requests focused and easy to review.
3. Include tests, validation notes, screenshots, or reproduction steps when they apply.
4. Update documentation when behavior, commands, or user-facing output changes.
5. A change may be merged only after approval by @GreyNOC or by a maintainer explicitly trusted by @GreyNOC. Codex-assisted review may be used when GreyNOC authorizes it.

Direct push access is not granted through comments, stars, forks, or pull requests. Contributors help through reviewable proposals unless GreyNOC grants a repository role separately.
